## 🛠️ Troubleshooting Common Issues

❌ **Loader Closes Immediately After Launch**
The most common cause is a missing .NET 6.0 runtime. The Cherax Mod Menu loader depends on it for its UI layer.
**Solution:** Download and install the latest `.NET 6.0 Desktop Runtime` from Microsoft, reboot, then relaunch.

❌ **Error 0x0012 on Injection**
This error indicates a timing conflict between the Cherax Mod Menu injector and a background anti-virus scanner.
**Solution:** Add the entire `C:\Tools\CheraxMenu` folder to your antivirus exclusion list and disable real-time scanning temporarily during injection.

❌ **Menu Overlay Not Appearing In-Game**
Usually caused by launching the game before the Cherax Mod Menu loader reaches the **Ready** state.
**Solution:** Close GTA 5, wait for the loader status bar to confirm **Ready**, then launch the game again and press `F5`.

❌ **Cherax Mod Menu Account Login Fails**
Expired session tokens can block authentication on the Cherax Mod Menu Site.
**Solution:** Clear the `cache` folder inside your install directory, restart the loader, and log in again with fresh credentials.

❌ **FPS Drop After Extended Sessions**
Older versions had a known memory leak in the protection module. The Cherax Mod Menu Enhanced `v3.1.0+` patched this.
**Solution:** Update to the latest build via the auto-updater. If already current, enable `reduce_hooks = true` in `config.cherax`.

❌ **Lua Scripts Not Loading**
Scripts must be placed in the directory defined by `script_dir` in config and must use `.lua` extension.
**Solution:** Verify the path in `config.cherax`, ensure the file encoding is `UTF-8`, and check the debug console for syntax errors.
